## 1 Введение

Данный документ описывает план тестирования приложения ["GameShelf"](https://github.com/dispronesson/game-shelf). Документ предназначен для людей, 
выполняющих тестирование данного проекта. Цель тестирования – проверка соответствия реального поведения программы проекта и ее ожидаемого поведения, 
которое описано в [требованиях](../SRS.md).

## 2 Объекты тестирования

Функциональные возможности, подлежащие проверке:

- отображение списка игр на главной странице;
- бесконечная прокрутка и подгрузка игр;
- поиск игр по названию;
- переход на страницу игры;
- просмотр описания игры, характеристик, платформ и системных требований;
- просмотр скриншотов;
- динамическое обновление результатов без перезагрузки страницы.

## 3 Атрибуты качества

1. Функциональность:
    - корректное взаимодействие с API;
    - поиск соответствует введённому запросу;
    - карточка игры содержит информацию и отображается корректно.
2. Юзабилити:  
    - интерфейс понятный и интуитивный;
    - полноэкранный просмотр легко закрывается (клик/ESC).
3. Производительность:  
    - плавная работа бесконечной прокрутки без подвисаний.

## 4 Риски

К приложению можно отнести следующие риски:

- API может вернуть неполные данные (нет системных требований, нет скринов);
- лимит запросов к RAWG API;
- медленный интернет увеличивает загрузку изображений;
- перегруз галереи большим количеством изображений;
- возможные изменения структуры API.

## 5 Аспекты тестирования

В ходе тестирования планируется проверить реализацию основных функций приложения. Аспекты, подвергаемые тестированию:
1. Главная страница:
    - загрузка первых игр при входе;
    - появление новых элементов при прокрутке;
    - корректное отображение карточек;
    - переход на страницу игры при клике на карточку с игрой. 
2. Поиск:
    - ввод + Enter изменяет результаты;
    - запрос обрабатывается без перезагрузки страницы;
    - query синхронизируется с URL;
    - поведение при пустом поиске.
3. Страница игры:
    - название;
    - платформы;
    - дата релиза;
    - жанры;
    - разработчики и издатели;
    - возрастной рейтинг;
    - описание;
    - системные требования (если есть).
4. Скриншоты:
    - скриншоты отображаются;
    - клик открывает fullscreen;
    - закрытие по клику на фон;
    - закрытие по Esc.
